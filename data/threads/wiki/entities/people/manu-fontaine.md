**Manu Fontaine**, founder and CEO of [Hushmesh](../orgs/hushmesh.md) (`Manu@hushmesh.com`). The SIG's most consistent architectural counterweight to Mark Novak — pushing for a fully trustworthy end state and recursive trust thinking.

## Recurring positions

| Position | Source |
|---|---|
| **Trustworthy Composability** — the composition mechanism must preserve every component's security strength | [^tc] |
| Verifiers and registries themselves must run in TEEs (recursive trust) | [^couplings] |
| Personal agents need per-entity keychains, not just one identity (the "ChatGPT logged into LinkedIn" example) | [^chatgptlinkedin] |
| Reference Architecture as North Star, then a guided migration path | [^couplings] |
| "Workload identity registry" — identity does not lie within the workload, but in a (TEE-resident) registry | [^couplings] |
| Provenance forces "chains of workloads" thinking — a mesh | [^prov] |
